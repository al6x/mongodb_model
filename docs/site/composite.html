Example of Composite Objects using Mongo Object Model.

Models are just ordinary Ruby Objects, so You can combine and mix it as You wish. The only differences are

  • main object has the _id defined.
  • child objects doesn’t have _id, but have _parent reference to the main object. and it’s children the _parent.

Callbacks, validations and conversions works on embedded objects the same way as on the main.

In this example we’ll create simple Blog and see how to embed Comment into Post.
